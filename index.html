RC-01: Sesión expirada sin renovación automática

CRÍTICO
287 casos (77.8% del total)
🔍 Descripción técnica del problema

El sistema no implementa refresh de tokens JWT/sesión. Cuando la sesión expira, el usuario es redirigido a login sin mensaje claro.

🖥️ Plataformas afectadas

Simón Web (146) · Simón Cotizadores (93) · SAI (44) · Navegador / Configuración (3)

📦 Productos afectados

Seguro de Autos (93) · Seguro de Cumplimiento (71) · General (54) · Ventas Generales (32)

🔧 Solución técnica de raíz

Implementar silent token refresh y configurar idle timeout con aviso previo de 5 min.

✅ Cómo garantizar que no se repita

Configurar refresh automático de sesión. Implementar interceptor HTTP que detecte 401/403 y renueve token sin interrumpir al usuario.

RC-03: Bug en motor de plantillas Tronador — concatenación incorrecta de textos y valores en PDF

ALTO
27 casos (7.3% del total)
🔍 Descripción técnica del problema

El motor de generación de documentos (Tronador) tiene un defecto en el renderizado de textos catalogados: concatena sin separadores, omite saltos de línea y calcula valores incorrectos en endosos de cancelación.

🖥️ Plataformas afectadas

Tronador / Simón Texto (22) · Simón Web (4) · Simón Cotizadores (1)

📦 Productos afectados

Banca + Movilidad (13) · General (6) · Seguro de Cumplimiento (3) · Colectivos / CH (2)

🔧 Solución técnica de raíz

Corregir el parser de textos catalogados en el módulo CB299236/CB299245. Agregar validación de integridad de valores monetarios antes de renderizar PDF. Implementar test unitario que valide formato de cada tipo de documento.

✅ Cómo garantizar que no se repita

Agregar suite de pruebas de regresión visual (screenshot diff) para cada tipo de documento generado. Ejecutar en pipeline antes de cada deploy. Implementar validación de schema en los textos catalogados al momento de guardarlos.

RC-02: Timeout/falla en microservicio de cotización sin circuit breaker

ALTO
3 casos (0.8% del total)
🔍 Descripción técnica del problema

El servicio de cotización falla silenciosamente cuando el backend tarda >30s. No hay circuit breaker ni fallback. El usuario ve error genérico "No se ha podido acceder al servicio web".

🖥️ Plataformas afectadas

Simón Cotizadores (3)

📦 Productos afectados

Seguro de Autos (3)

🔧 Solución técnica de raíz

Implementar circuit breaker (Resilience4j/Hystrix) con fallback que muestre mensaje de reintento. Configurar timeout de 10s con retry automático x2. Agregar health check en Datadog con alerta a <5 min.

✅ Cómo garantizar que no se repita

Agregar monitor sintético en Datadog que ejecute cotización completa cada 5 min. Configurar alerta P2 si falla 2 veces consecutivas. Implementar degraded mode que permita cotización offline básica.
